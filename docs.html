Écosystème Tommy Burger — POS & Mobile

PROJET PROFESSIONNEL JAVA / JAVAFX FLUTTER / DART MYSQL API 2026

Vision & Problématique Métier

Dans l'industrie de la restauration rapide, la précision et la rapidité sont les piliers de la rentabilité. Tommy Burger présentait un besoin double : une interface de vente ultra-réactive au comptoir et une solution de gestion mobile pour le gérant afin de suivre l'activité en temps réel sans être présent physiquement.

Architecture Multi-Plateforme

L'écosystème repose sur une base de données MySQL centralisée. Le logiciel Desktop communique directement avec la DB, tandis que l'application mobile (développée sous Flutter) consomme les données via une API REST sécurisée, permettant une consultation fluide des ventes sur Android et iOS.

Détails Techniques de l'Implémentation

  • Interface Desktop (JavaFX) : Création d'un système de panier dynamique avec gestion des menus et suppléments en temps réel.
  • Pilotage Hardware : Utilisation de bibliothèques de bas niveau pour piloter directement les imprimantes thermiques ESC/POS via USB/Série.
  • Ouverture Tiroir : Envoi de codes hexadécimaux spécifiques au port de l'imprimante pour déclencher l'ouverture physique du tiroir-caisse.
  • App Mobile Admin : Suivi du chiffre d'affaires, gestion des stocks et alertes de rupture en temps réel via Flutter.

Panneau d'Administration & Reporting

Une interface dédiée permet au gérant de modifier les prix, d'ajouter des produits, de gérer les accès du personnel et d'extraire des rapports de ventes quotidiens exportables.

BarakaFood — Plateforme Solidaire

WEB APPLICATION PHP / MYSQL FULL-STACK GITHUB

Vision du Projet & Objectif Social

BarakaFood est né d'un constat simple : des tonnes de nourriture sont jetées chaque jour par les restaurateurs alors que des besoins alimentaires subsistent. Cette plateforme agit comme un intermédiaire numérique sécurisé pour faciliter le don alimentaire.

Spécifications Techniques Web

Le projet a été développé en utilisant le stack LAMP. Le backend en PHP gère toute la logique de mise en relation, tandis que MySQL assure le stockage des données sensibles (utilisateurs, annonces de dons, géolocalisation des restaurants).

Gestion Collaborative & DevOps
  • Travail d'équipe coordonné entre 4 développeurs.
  • Utilisation intensive de Git et GitHub pour la gestion des branches et la résolution des conflits de code.
  • Vidéo de démonstration produite pour présenter le parcours utilisateur (UX) sur YouTube.

Fonctionnalités Clés

  • Système d'inscription à double profil : Donateur (Restaurant) et Bénéficiaire (Association/Individu).
  • Système de recherche par ville et par type de nourriture disponible.
  • Messagerie interne pour coordonner la récupération des surplus.

Gestion de Cabinet Médical

DESKTOP APP JAVA SWING MS ACCESS LAN SYNC

Numérisation du Parcours Patient

Ce logiciel remplace intégralement les dossiers papier par une base de données sécurisée. Il permet de suivre le patient de son arrivée en salle d'attente jusqu'à la prescription finale.

Défis Techniques : Réseau & Localisation

L'enjeu majeur était la synchronisation en temps réel sur le réseau local (LAN). La secrétaire doit pouvoir marquer un patient comme "arrivé" et cette information doit apparaître instantanément sur l'écran du médecin dans son bureau via une base MS Access partagée.

Modules Inclus

  • Module Secrétariat : Prise de rendez-vous, création de fiches, gestion de la file d'attente.
  • Module Médecin : Historique médical complet, gestion des antécédents, saisie des ordonnances.
  • Sécurité : Accès protégé par identifiant et mot de passe avec différents niveaux de permissions.

L'objectif de ce projet académique est de transformer les données brutes d'une entreprise en indicateurs stratégiques. Le système analyse plusieurs critères pour recommander la meilleure option possible aux dirigeants.

Application de Chat (Sockets TCP)

NETWORKING JAVA TCP/IP MULTI-THREADING

Architecture Réseau

Développement d'une messagerie instantanée basée sur l'architecture Client-Serveur. Ce projet démontre la maîtrise des flux de données bidirectionnels sur le réseau TCP/IP.

Gestion de la Concurrence

Le serveur est capable de gérer plusieurs dizaines de clients simultanément grâce au Multi-threading. Chaque connexion est isolée dans un thread dédié pour éviter tout blocage du service.

  • Utilisation des Sockets Java pour la communication fiable.
  • Gestion des évènements de connexion, déconnexion et erreurs réseau.
  • Système de broadcast pour envoyer des messages à tous les utilisateurs.
